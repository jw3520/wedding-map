# Wedding Map - MVP Specification

## 1. 개요 (Overview)
**Wedding Map**은 결혼 준비 과정의 정보 비대칭을 해소하고, 예비부부들이 플래너 없이도 스스로 일정을 관리할 수 있도록 돕는 **능동형 웨딩 타임라인/체크리스트 PWA**입니다.
첫 MVP(최소 기능 제품)는 '예산 관리'보다는 **'전체적인 로드맵 제공과 체크리스트(리마인드)'** 그리고 **'쉬운 공유(Read-only 대시보드)'**에 집중합니다.

## 2. 핵심 목표 (Core Objectives for MVP)
1. **로드맵 및 스마트 체크리스트 (Timeline & Checklist)**
   - 예식일을 기준으로 역산된 타임라인과 필수/선택 할 일 목록 제공.
   - 각 할 일의 필요성과 시기에 대한 명확한 이유 안내 (가이드/툴팁).
2. **바이럴 및 파트너 참여 유도 (Read-only Sharing)**
   - 앱 설치나 가입 없이도 진행 상황을 볼 수 있는 공유 링크 제공.
   - 소극적인 파트너나 양가 가족에게 공유하여 자연스러운 서비스 인지 및 활성 사용자 전환 유도.

## 3. 주요 기능 명세 (Key Features)

### 3.1 D-Day 기반 로드맵 & 체크리스트
- **초기 설정:** 사용자가 '예식 예정일(또는 목표월)'을 입력하면 자동으로 전체 준비 로드맵(타임라인) 생성.
- **체크리스트 뷰:** 
  - 남은 기간별(예: D-365, D-180, D-90, D-30 등) 필수/선택 할 일(Task) 구분 노출.
  - 완료(Done), 진행 중(In Progress), 예정(Todo) 상태 표시.
- **정보성 툴팁 (Why & When):**
  - *예: "웨딩 촬영(스튜디오)은 최소 예식 6개월 전에 해야 합니다. (이유: 촬영 후 사진 셀렉과 보정에 시간이 소요되며, 최종본이 나와야 모바일 청첩장을 제작할 수 있기 때문입니다.)"*
- **(추후 고도화 예정):** 유저 데이터가 쌓이면 "예비부부의 XX%가 이 항목을 진행했어요"와 같은 통계 데이터 노출.

### 3.2 능동형 리마인드 (알림)
- 각 타임라인의 마지노선이 다가오면 PWA 푸시 알림 또는 서비스 내 배너로 리마인드 제공.
- 예: "다음 달까지 스튜디오 예약을 확정하는 것이 좋습니다!"

### 3.3 진행 상황 대시보드 및 공유 (Read-only URL)
- **커플 대시보드:** 전체 진행률(Progress Bar), 가장 시급한 다음 할 일(Next Up) 요약.
- **공유 링크 생성:**
  - 고유한 URL(예: `weddingmap.app/c/couple-id`) 생성.
  - 해당 링크 접속 시 수정 권한은 없지만 전체적인 로드맵, 확정된 일정(식장, 스튜디오 날짜 등), 현재 진행률을 깔끔한 UI로 열람 가능.
- **기대 효과:** 상대적으로 관심이 적은 배우자가 링크를 통해 진행 상황을 보다가 직접 참여하게 되거나, 양가 부모님께 현황을 브리핑하는 용도로 활용됨.

## 4. 제외된 기능 (Out of Scope for MVP)
- **상세 예산 관리 / 가계부 기능:** 엑셀을 대체하는 복잡한 지출 관리 기능은 2차 페이즈로 연기. (단, 각 태스크의 예상 비용을 참고용으로만 노출하는 것은 고려 가능)
- **외부 업체 제휴 및 예약 결제:** 플랫폼 내에서 스드메 업체를 직접 검색/결제하는 기능. (순수 일정 관리 도구로 포지셔닝)

- 화면 흐름도(User Flow) 및 UI 레이아웃 초안 기획.
