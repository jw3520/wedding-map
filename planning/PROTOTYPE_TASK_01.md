# Codex Task 01: Build MVP Prototype for "Wedding Map"

## 1. Goal (작업 목표)
2030 예비 신혼부부를 타겟으로 하는 모바일 우선(Mobile-first) 웹 앱 'Wedding Map'의 MVP 프로토타입을 개발합니다. 
시장의 반응을 빠르게 확인하기 위한 목적이므로, **목업(Mock) 데이터**를 적극 활용하여 핵심 기능인 **'D-Day 기반 로드맵'**과 **'공유용 Read-only 대시보드'**가 실제로 작동하는 것처럼 보이게 트렌디한 UI/UX로 구현하는 것이 최우선 목표입니다.

## 2. Design Concept & UI/UX (2030 타겟)
- **모던 & 미니멀리즘:** 토스(Toss)나 핀터레스트처럼 여백이 넉넉하고, 텍스트의 위계(Typography)가 뚜렷하며 직관적인 UI.
- **모바일 네이티브 감성:** PWA로 동작할 때 네이티브 앱처럼 느껴지도록 부드러운 화면 전환, 바운스 효과, 카드(Card) 기반의 컴포넌트를 사용합니다.
- **감성적 터치:** 너무 딱딱한 To-do 리스트가 아닌, 결혼 준비의 설렘을 줄 수 있는 부드러운 컬러 팔레트(예: 파스텔 톤 포인트 컬러, 둥근 모서리)를 적용합니다.

## 3. Tech Stack (권장 사항)
- **Framework:** React (Vite) + TypeScript.
- **Styling:** Vanilla CSS (CSS Modules)를 우선하여 깔끔하게 작성. (2030 감성에 맞는 미려한 디자인을 위해 Framer Motion 같은 애니메이션 라이브러리 추가 권장).
- **State Management:** 초기 단계이므로 Context API + `localStorage`를 사용하여 새로고침해도 예식일과 체크리스트 상태가 유지되도록 구현.
- **PWA:** `manifest.json` 추가로 모바일 홈 화면 설치가 가능하도록 구성.

## 4. Core Features to Implement
1. **온보딩 뷰 (Onboarding)**
   - 앱 진입 시 "결혼 예정일이 언제인가요?"를 묻는 감각적인 날짜 선택 화면.
   - 날짜를 선택하면 다음 화면으로 부드럽게 넘어감.
2. **타임라인 & 체크리스트 (Main View)**
   - 예식일을 기준으로 역산된 카테고리(예: D-365, D-180, D-90)별 할 일 목록을 렌더링.
   - 각 할 일(Task) 카드를 터치하면, "왜 이 시기에 해야 하는지" 설명하는 상세 툴팁/아코디언이 펼쳐짐.
   - 완료(Done) 체크박스 토글 기능.
3. **공유용 대시보드 (Read-only View)**
   - 하단 탭이나 상단 버튼을 통해 '공유 뷰'로 전환.
   - 전체 준비 달성률(%)을 보여주는 예쁜 프로그레스 바.
   - 완료된 핵심 일정과 앞으로 다가올 가장 시급한 일정(Next Up)을 요약해서 보여줌.
   - 파트너나 부모님이 볼 수 있도록 체크/수정 기능이 비활성화된 깔끔한 브리핑 화면.
